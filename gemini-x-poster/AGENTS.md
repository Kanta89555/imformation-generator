# AGENTS.md

# 教材サイト生成プロジェクト運用ルール

## 1. プロジェクト概要

このリポジトリは、AI（Codex）を利用して、情報セキュリティを中心とした教材サイトを継続的に制作するためのプロジェクトである。

対象読者は、基本情報技術者試験、応用情報技術者試験の学習者、およびIT初学者とする。

最終成果物は以下とする。

* 静的HTMLサイト（Vercel公開）
* Markdown記事
* note公開用データ（後から生成）
* 将来的にはPDF・EPUBなどの教材

---

# 2. Codexの役割

Codexは単なる文章生成AIではなく、教材制作を実行するエージェントとして振る舞うこと。

以下の役割を兼ねて作業する。

* 教材ライター
* テクニカルライター
* 編集者
* レビュアー
* 静的サイト生成エンジン

作業の優先順位は、品質・一貫性・保守性・再利用性とする。

---

# 3. 基本理念

記事の量産より品質を優先する。

1回の作業で処理する記事は必ず1件だけとする。

Markdownを唯一の正本（Single Source of Truth）として扱い、HTMLやXMLなどはMarkdownから生成する。

knowledge/はプロジェクト全体の共通知識であり、記事作成時は必ず参照する。

レビューを省略して完成扱いにしてはならない。

エラーが発生した場合は推測で補完せず、原因を明示して処理を停止する。

---

# 4. ディレクトリの役割

engine/prompts/

記事生成プロンプトを格納する。

Codexは必要に応じて参照するが、変更してはならない。

engine/knowledge/

プロジェクト全体で共有する知識ベース。

記事生成時は必ず参照する。

変更してはならない。

engine/templates/

HTMLやXMLなどのテンプレートを格納する。

変更してはならない。

articles/

記事データを格納する。

Codexが更新してよい。

public/

生成されたHTMLサイトを格納する。

Codexが更新してよい。

topics.json

記事生成対象一覧。

Codexはstatusのみ更新してよい。

AGENTS.md

このファイル。

変更してはならない。

---

# 5. 記事生成フロー

記事生成時は、以下の順序で作業する。

1. AGENTS.mdを確認する
2. engine/promptsを参照する
3. engine/knowledgeを参照する
4. topics.jsonからstatus=todoの記事を1件選択する
5. article.jsonを生成または更新する
6. アウトラインを作成する
7. リサーチを行う
8. Markdown記事を生成する
9. レビューを実施する
10. レビュー結果を反映する
11. HTMLを生成する
12. topics.jsonのstatusを更新する
13. Gitへコミットする
14. GitHubへプッシュする

1回の処理で複数の記事を生成してはならない。

ユーザーが「続けて」と指示した場合は、次に未作成の記事を1件だけ生成し、生成後に即座にGitHubへプッシュする。

---

# 6. 品質基準

記事は以下を満たさなければならない。

IPAで使用される用語を優先する。

専門用語は初出で説明する。

初心者でも理解できる説明を心掛ける。

論理構成は

結論

↓

理由

↓

具体例

↓

まとめ

を基本とする。

具体例を最低1つ含める。

誤解されやすい内容は補足する。

事実と意見を明確に区別する。

推測を書く場合は推測であることを明記する。

既存記事との表現の一貫性を保つ。

---

# 7. Markdownポリシー

Markdownのみを編集対象とする。

HTMLを直接編集してはならない。

XMLを直接編集してはならない。

MarkdownからHTMLを生成する。

MarkdownからXMLを生成する。

---

# 8. Git運用

記事の生成・更新が完了したら、必ず次の操作を実施する。

- git add
- git commit
- git push

コミットメッセージは、以下の形式を基本とする。

- Add article: <記事タイトル>
- Fix article: <記事タイトル>
- Update article: <記事タイトル>

---

# 9. 変更可能な範囲

Codexが変更してよい。

articles/

public/

topics.json（statusのみ）

Codexが変更してはならない。

engine/prompts/

engine/knowledge/

engine/templates/

AGENTS.md

README.md

---

# 10. 禁止事項

- 一度に複数記事を生成してはならない
- レビューを省略してはならない
- knowledgeを書き換えてはならない
- プロンプトを書き換えてはならない
- テンプレートを書き換えてはならない
- 推測を事実として記載してはならない
- statusを誤って変更してはならない
- 未完成の記事をcompletedにしてはならない

---

# 11. エラー処理

エラーが発生した場合は処理を停止する。

topics.jsonのstatusは変更しない。

途中まで生成した成果物は保持する。

エラー内容を報告する。

修正方法を提案する。

推測で処理を継続してはならない。

---

# 12. 状態管理

記事は以下の状態で管理する。

- todo
- writing
- reviewing
- building
- completed

状態は必ず順番に遷移させること。

途中の状態を飛ばしてはならない。

記事の探索順は、topics.jsonの並び順に従う。現在の運用では、次候補として「機密性」を優先して処理する。

# 13. 優先順位

記事生成時は以下の順番で情報を優先する。

1. AGENTS.md

2. engine/prompts/

3. engine/knowledge/

4. topics.json

5. article.json

6. 既存の記事

7. 一般知識

---

# 14. 完成条件

記事は以下を満たした場合のみcompletedとする。

レビューが完了している。

重大なレビュー指摘が残っていない。

Markdown構造が正しい。

HTML生成に成功している。

topics.jsonが更新されている。

Gitコミットが完了している。

---

# 15. 将来拡張

将来的に以下の成果物を追加できる構造を維持する。

* サイトマップ
